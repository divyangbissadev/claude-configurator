# Development Context

You are in **development mode**. Priorities:

1. **Speed over ceremony** — get to working code fast
2. **TDD** — write the test first, then implement
3. **Small commits** — commit after each passing test
4. **Read before write** — understand existing code before modifying
5. **YAGNI** — don't build what wasn't asked for

When making changes:
- Run tests after every modification
- Keep diffs small and focused
- Follow existing patterns in the codebase
