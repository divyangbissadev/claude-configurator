# Documentation Conventions

- CLAUDE.md is the source of truth for AI agents — keep it current
- One-line docstrings unless explaining non-obvious business logic
- Comments explain WHY, never WHAT — the code explains what
- No commented-out code — use version control history
- ADRs for architectural decisions that aren't obvious from the code
- README for human onboarding, CLAUDE.md for AI onboarding
