# Test Coverage

## Steps

1. **Run coverage**: Execute the project's coverage command
2. **Parse report**: Identify files below the coverage threshold
3. **Prioritize gaps**: Critical paths > utilities > edge cases
4. **Suggest tests**: For each uncovered path, describe what test would cover it
5. **Report**: Coverage percentage per module, list of gaps with suggested tests
