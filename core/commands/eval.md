# Eval

## Steps

1. **Define criteria**: What does success look like? (from spec, ticket, or user)
2. **Run checks**: Tests, lint, type-check, coverage
3. **Manual verification**: Does the implementation match the spec?
4. **Quality assessment**: Code quality, test quality, documentation completeness
5. **Score**: Rate each criterion (pass/fail/partial)
6. **Report**: Structured evaluation with scores and actionable feedback
