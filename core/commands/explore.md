# Explore

Systematic codebase exploration for understanding before making changes.

## Steps

1. **Project overview**: Read CLAUDE.md, README, package.json/pyproject.toml
2. **Directory structure**: Map top-level dirs and their purposes
3. **Entry points**: Find main execution paths (main.py, index.ts, etc.)
4. **Architecture**: Identify patterns (MVC, hexagonal, pipeline, etc.)
5. **Key abstractions**: List base classes, interfaces, core types
6. **Conventions**: Code style, testing patterns, commit format
7. **Dependencies**: Major libraries and their roles
8. **Report**: Structured summary of findings
