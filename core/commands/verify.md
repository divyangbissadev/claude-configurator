# Verify

## Steps

1. **Re-read requirements**: What was asked for? What are the acceptance criteria?
2. **Run tests**: Full test suite — all must pass
3. **Manual check**: Verify the feature works as described (not just that tests pass)
4. **Edge cases**: Test boundary conditions, error paths, empty states
5. **Regression check**: Verify no existing functionality broke
6. **Report**: Pass/fail with evidence for each acceptance criterion
